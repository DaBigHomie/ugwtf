# UGWTF — 8-Phase Copilot Approval Pipeline

Generated by `src/generators/copilot-automation.ts`. Deploys as GitHub Actions YAML to target repos.

## Pipeline Overview

```
Issue labeled → Phase 1 → ... → Phase 8
                 ↓ fail
              Phase 7 (re-assign, self-heal)
```

## Phases

| Phase | Name | Trigger | Action |
|-------|------|---------|--------|
| 1 | Issue Triage & Assign | Issue labeled `automation:copilot` | Check existing PRs → assign Copilot → create branch + draft PR |
| 2 | Draft → Ready | Draft PR created | Promote to ready-for-review → request Copilot code review |
| 3 | Quality Validation | PR opened/synced | `npx tsc --noEmit` + `npm run lint` + `npm run build` + DB migration detection |
| 3.5 | E2E Validation | After Phase 3 pass | `npx playwright test` — **non-blocking** (advisory only) |
| 4 | Review Feedback | Copilot review posted | Implement Copilot suggestions → re-push → re-trigger Phase 3 |
| 5 | Auto-Merge | All checks pass | Auto-approve + squash merge (**blocked for DB migration PRs**) |
| 6 | DB Migration Firewall | `.sql` or `supabase/migrations/` detected | Block auto-merge → post manual steps → require human merge |
| 7 | Failure Handling | Any phase fails | Re-assign Copilot → self-healing retry loop |
| 8 | Post-Merge Cleanup | PR merged | Update linked issues → chain to next issue → update scoreboard |

## DB Migration Firewall (Phase 6)

When a PR touches migration files, Phase 5 auto-merge is blocked. The pipeline posts manual steps:

1. Apply migration SQL via **Supabase Dashboard → SQL Editor**
2. Regenerate types:
   ```bash
   npx supabase gen types typescript --project-id <ID> > <types-path>
   ```
3. Deploy Edge Functions if needed
4. Run quality gates: `npx tsc --noEmit && npm run lint && npm run build`
5. Merge manually after verification

**Per-repo config** provides: `supabaseProjectId`, `supabaseTypesPath`, `e2eCommand`, `defaultBranch`, `nodeVersion`.

## PR Review Agent Details (`pr-agents.ts`)

### PR Copilot Reviewer (`pr-review`)
- Filters PRs by: `automation:copilot` label OR `copilot` author OR `copilot/` branch prefix
- Detects DB migrations: checks `supabase/migrations/` paths and `.sql` extensions
- Posts review table: Changed Files | DB Migration? | Destructive Ops? | Auto-Merge?
- If DB migration → adds `database` + `needs-review` labels

### PR Batch Processor (`pr-batch-process`)
- Scans ALL open PRs (not just Copilot)
- Flags stale drafts → adds `stalled` label + comment
- Ensures `automation:copilot` label on `copilot/*` branch PRs

### PR Completion Tracker (`pr-completion-tracker`)
- Monitors merged Copilot PRs
- Extracts linked issues: `Fixes #N`, `Closes #N`, `Resolves #N`
- Transitions labels: `automation:in-progress` → `automation:completed`
