# Prompt Chain System Reference

> Source of truth for the prompt chain pipeline. Synced to repos as `.github/instructions/chain-instructions.md`.

## Overview

The prompt chain is UGWTF's **primary e2e pipeline**:

```
.prompt.md files → prompt-chain.json → GitHub Issues → Copilot PRs → Production
```

Each step is owned by a UGWTF command:
1. **`prompts`** — Scans `.prompt.md` files, validates, creates spec issues
2. **`generate-chain`** — Builds `prompt-chain.json` from scanned prompts
3. **`chain`** — Creates chain-tracker issues, assigns Copilot, advances the chain
4. **`issues`** — Triages issues, detects stalled work, re-assigns
5. **`prs`** — Reviews Copilot PRs, enforces DB migration firewall

## Prompt Files

Prompt specs live in the repo as `.prompt.md` files with YAML frontmatter:

```yaml
---
title: "Design database schema"
priority: p0
scope: db
type: feat
wave: 1
dependencies: []
---

## Objective
Design and implement the Supabase database schema...

## Success Criteria
- [ ] All tables created with proper RLS
- [ ] Types generated and committed
```

## prompt-chain.json

Generated by `ugwtf generate-chain`. Maps prompts to a sequenced chain:

```json
[
  {
    "position": 1,
    "promptFile": "prompts/001-database-schema.prompt.md",
    "issueNumber": 204,
    "wave": 1,
    "severity": "critical",
    "labels": ["database", "priority:p0"],
    "status": "open"
  }
]
```

## Issue Layers

The chain creates **two types of issues**:

### Spec Issues (label: `prompt-spec`)
- Full implementation prompt with objectives, criteria, code examples
- Created by `ugwtf prompts`
- Title format: `feat(db): Design database schema [FI-01]`

### Chain Issues (label: `chain-tracker`)
- Sequencing/orchestration: position, wave, dependencies
- Created by `ugwtf chain`
- Title format: `feat(db): Design database schema [FI-01]`
- These are the issues Copilot gets assigned to

## Chain Advancement

When a Copilot PR merges:
1. The `prompt-chain.yml` workflow fires
2. Reads `prompt-chain.json` to find the next issue in sequence
3. Adds `copilot:ready` label to the next issue
4. Copilot picks up the next issue

### Known Issue: Copilot Assignment
- `addAssignees({assignees: ['copilot']})` does NOT work
- Use GitHub's native "Assign to Copilot" (UI or MCP tool)
- The workflow adds labels but manual assignment may be needed

## Wave System

Issues are grouped into waves for parallel execution:
- **Wave 1**: Foundation (database, auth, design system) — sequential
- **Wave 2**: Core pages — can run in parallel within wave
- **Wave 3**: Features — can run in parallel
- **Wave 4**: Polish — can run in parallel

Within a wave, issues respect dependency order from `prompt-chain.json`.

## Commands

```bash
# Scan prompts, validate frontmatter
ugwtf prompts <alias>

# Generate prompt-chain.json from scanned prompts
ugwtf generate-chain <alias>

# Create chain issues, assign Copilot to first unstarted
ugwtf chain <alias>

# Check for stalled issues, re-triage
ugwtf issues <alias>

# Review Copilot PRs, enforce quality
ugwtf prs <alias>
```

## Troubleshooting

### Chain not advancing after PR merge
- Check if `prompt-chain.yml` workflow ran
- Verify the merged PR had `prompt-chain` label
- Check `prompt-chain.json` for correct issue numbers

### Copilot not picking up assigned issue
- Verify Copilot coding agent is enabled on the repo
- Use MCP tool `github-assign_copilot_to_issue` instead of label-based assignment
- Check if issue body has enough context for Copilot to work with

### Issues created without proper labels
- Run `ugwtf issues <alias>` to auto-triage
- Run `ugwtf labels <alias>` if labels are missing from the repo
